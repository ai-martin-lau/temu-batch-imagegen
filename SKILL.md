# TEMU 商品图批量出图 · SKILL

把 `input/` 里的【每一个输入单元】+【按名字匹配到的中文 prompt】，用 codex 内置 `image_gen` 各扩成一整套电商主图 / 套图。
一个**输入单元**是二者之一：
- **单张图片**（`input/连衣裙.png`）→ 1 张参考图出一整套；
- **一个文件夹（组图）**（`input/连衣裙/`）→ 夹内多张图是**同一个商品**的多角度参考，全部塞进**同一个会话**当多张 `-i` 参考图，综合参考出一整套。

**零额外成本**：走 codex 订阅，免 API key；图先落 codex 默认目录，再拷进本项目 `output/`。

> ⚠️ `image_gen` 是 **codex 的内置工具**，走 codex 订阅、**免 API key**。出图脚本 `batch.sh` 已写好，直接跑即可（input/ 放单图、组图文件夹、混着放都行），无需你再写代码。

## 运行流程（codex 按此执行）

### 铁律 0 · 不要问问题，自动处理 input/ 里的全部单元

**不要**问"用哪个 / 用哪个 prompt / 出几张"——这三件都已确定，直接开干：

- **处理哪些单元**：**只认根目录 `input/` 直下**的每一项，一个都不漏，每个各出一套：
  - **单张图片** = 一个单元（1 张参考图）；
  - **一个文件夹** = 一个单元（组图：夹内多张图是**同一商品**的多角度参考，全部进**同一会话**，不要拆成多个单元、不要每张各出一套）。
  - ⚠️ `example/` 仅供参考/照抄，**绝不作为输入处理**；只扫根目录的 `input/` 和 `prompt/`，不要碰 `example/input`、`example/prompt`。
- **用哪个 prompt**：按下面《prompt 匹配规则》**按单元名字（文件夹名 / 图片名）自动匹配**，不让用户选。
- **出几张 / 什么尺寸**：完全由**匹配到的 prompt 文本写明**（如"输出 6 张 3:4 + 1 张 1000×1000"），照它做；**prompt 没写就不要自行假设固定尺寸/张数**，按其文字意图合理处理。

### 推荐做法 · 直接跑 batch.sh

最稳的方式是直接执行批量脚本，它已实现隔离 + 并发 + 失败回退：

```bash
bash batch.sh            # 默认最多 3 个 codex 进程并发
MAX=5 bash batch.sh      # 想更快就调大并发（注意 codex 订阅可能限流）
```

它做的事：

- **一单元一进程 = 一个独立 codex 会话**：不同商品互不干扰（隔离），多进程同时跑（并行）。
  - 同一单元（含组图文件夹里的多张参考）在**同一会话内**生成——共享同一组参考图，保证同商品多张产出之间一致，这是想要的。
  - 不同商品之间**绝不**塞进同一会话，否则会飘色 / 串款。
- **并发 3，失败回退串行**：先 3 个一波并发；失败（进程崩溃 / 零产出）的单元收集起来，最后一个个串行重试——**每次只清空该商品自己的目录后整组重出，不影响其它商品**。
- 输出落 `output/batch-<时间戳>/<单元名>/01.png 02.png …`，日志在该目录的 `.logs/`。

### prompt 匹配规则（每个单元按【名字】找 prompt，统一 `.txt`）

名字 = **文件夹名**（组图）或 **不带扩展名的图片名**（单图）。

1. **同名优先**：`prompt/<名字>.txt`（如 `input/连衣裙.png` 或 `input/连衣裙/` → `prompt/连衣裙.txt`）。
2. **默认兜底**：没同名时用 `prompt/默认.txt`（通用 prompt 约定命名为 `默认.txt`）。
3. **单文件兜底**：`prompt/` 里只有一个 `.txt` 时，所有单元都用它（名字随便）。
4. 都没命中 → 跳过该单元并告警。

### 每张图内部的铁律（codex 在每个会话里必须遵守）

- **每一张都单独一次 `image_gen` 调用**：不要把多张拼到一张画布，不要用 `n` 代替"不同的图"（`n` 只是同一 prompt 的相似变体、且共用一个尺寸）。
- prompt 要几张就调用几次，**每次产出一张不同的图**（不是近似复制）；至于怎么不同、要不要变姿势/场景/尺寸——**完全以 prompt 写明的为准**，不在此规定。
- **严格按 prompt 写明的要求执行**（颜色、版型、露脸、背景、品牌、文案、尺寸、风格等都在 prompt 里）——不自行增删，**不在此硬编码任何业务/创意规则**。
- **边生成边存**：每生成好一张立刻拷到本商品的 `output/batch-*/<单元名>/`，命名 `01.png 02.png …`（两位补零），打印绝对路径——进度可见、中断也不丢。

### 验尺寸 + 自检 + 交付

- **先验尺寸（必做）**：内置 image_gen 不严格遵守指定尺寸（常把竖图出成 9:16）。每张生成后**立即按 prompt 写明的尺寸核对**——prompt 要什么比例/像素就核对什么（`sips -g pixelWidth -g pixelHeight <文件>` 读宽高）。**尺寸标准一律以 prompt 为准，不要假设固定值（不要默认就是 3:4 或 1:1）。**
  - 尺寸不对的那张：**重新生成**该张（至多 2-3 次）；多次仍不对就如实报告，不做裁切兜底。
- 逐张 **看图自检**：是否满足 prompt 写明的各项要求（尺寸、内容、文案、风格等一律以 prompt 为准）、文字有没有渲染出错——不满足的**只重生那一张**，不用整套重来。
- 结束时报告：本次 `output/batch-*/` 路径 + 每个商品实际产出张数 + 各自用的哪个 prompt + 失败/跳过的图。

## 坑 & 约定（务必遵守）
1. `image_gen` 是 codex 内置工具，**别用 CLI / 别要 API key / 别用 SVG·HTML 占位代替**。
2. **不要问用户问题**：input/ 全处理、prompt 按名匹配、张数看 prompt——三件都已确定。
3. **每张单独一次调用**——多张图靠多次调用，不是靠 `n`（`n` 只用于同一 prompt 的变体）。
4. **参考图锁一致性**：每个 codex 会话始终把本单元的参考图作为 `-i`（组图文件夹的多张参考全给这一个会话）；不同商品**必须各自独立会话**，否则同商品多张会"飘色 / 变款"、不同商品会互相串。
5. **一切业务约束以 prompt 为准**：露脸 / 背景 / 品牌 logo / 文案等具体要求都写在 prompt 里，按 prompt 执行——**不在本文件硬编码业务规则**（如"必须去 logo"），prompt 怎么说就怎么做。
6. **图像模型渲染文字有局限**：渲染文字（尤其外语）可能出错字——这是模型局限、不是 prompt 的错；自检发现就重生该张，对要求 100% 准确的文字可先出纯图、后期用代码精确叠字。
7. **边生边存**：别等全部做完才存，省得中途中断全丢。
8. **codex CLI 调用细节**（写脚本时）：`-i/--image` 是可变参数，会把后面的 prompt 也当成图片吃掉；prompt 要走 **stdin 管道**喂入，`-i` 后只留图片。参见 `batch.sh`。
