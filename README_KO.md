# TEMU 상품 이미지 일괄 생성 도구

`input/` 에 상품 입력을 한 번에 넣기만 하면 됩니다. 각 상품은 **이미지 한 장**이거나 **폴더(같은 상품의 여러 각도 참조 이미지)** 일 수 있습니다. 매칭된 창작 프롬프트에 따라, **codex 내장 `image_gen`**(당신의 codex 구독으로 동작, **API 키 불필요**)으로 상품마다 N 장의 이커머스 메인 이미지 / 이미지 세트를 일괄 생성해 `output/` 에 저장합니다.

용도: 실제 촬영한 상품 사진들을 지정한 스타일로 TEMU / 해외직구 / 온라인 스토어용 메인 이미지 + 이미지 세트로 확장.

---

## 1. 준비 (최초 1회)

**로그인된 codex CLI** 만 있으면 됩니다(codex 구독이면 충분):

```bash
codex --version      # 버전이 출력되면 OK
codex login          # 미로그인 시 안내에 따라 로그인
```

Python, `OPENAI_API_KEY`, 추가 설치 모두 불필요합니다.

---

## 2. codex 에 연결

`temu-batch-imagegen/` 폴더 전체를 codex 의 skills 디렉터리에 넣습니다:

```bash
cp -R temu-batch-imagegen ~/.codex/skills/
```

> skills 디렉터리에 넣지 않고 이 폴더로 `cd` 해서 바로 써도 됩니다("방법 B" 참조).

---

## 3. 사용법

> 핵심 규칙: **`input/` 안의 모든 단위(단일 이미지 + 이미지 폴더)가 처리됩니다. "어느 것을 / 어떤 프롬프트로 / 몇 장" 을 묻지 않습니다** —— 이 세 가지는 이미 정해져 있습니다(전부 처리, 이름으로 프롬프트 매칭, 장수는 프롬프트 본문 참조).

### 가장 간단: `批量出图.command` 더블클릭 (비개발자용)
`input/` 에 이미지를 넣고 **`批量出图.command` 더블클릭** 만 하면 됩니다 —— 터미널이 자동으로 열리고, 완료 후 `output/` 폴더가 표시됩니다.
- `input/` 이 비어 있으면 안내 후 input 폴더를 엽니다. codex 미설치 / 미로그인 시 명확히 오류를 냅니다.
- 첫 더블클릭에서 macOS 가 차단하면 **우클릭 → 열기** 로 한 번 허용하세요.
- 전제는 동일하게 **codex 설치 + 로그인**.

### 방법 A: 한 줄 명령 일괄 (터미널 사용자 권장)
1. **입력**을 `input/` 에 둡니다. 두 형태 혼합 가능:
   - **단일 이미지**(`dress.png`) = 한 상품, 참조 이미지 1 장;
   - **폴더**(`jeans/`, 같은 상품의 여러 사진) = 한 상품의 **이미지 세트**. 내부 여러 장이 다각도 참조로 같은 세션에 전달됩니다.
2. 프롬프트 준비(`prompt/*.txt`, 요구사항을 자연어로 작성): 범용은 `默认.txt`. 특정 상품 커스텀은 **이미지 이름 또는 폴더 이름과 동일한** `.txt` 생성.
3. 이 폴더에서 실행:

```bash
bash batch.sh            # 기본 최대 3 병렬
MAX=5 bash batch.sh      # 더 빠르게 하려면 병렬 수 증가(codex 레이트 제한 주의)
```

- **이미지 1 개당 프로세스 1 개**: 각 이미지가 독립 codex 세션을 가져 서로 간섭하지 않습니다. 기본 3 병렬, **실패 시 자동으로 직렬 재시도로 폴백**.
- **프롬프트 자동 매칭**: 각 단위는 **이름**(이미지 / 폴더 이름)으로 `prompt/<이름>.txt` 를 찾습니다. 동명이 없으면 `prompt/默认.txt`. `prompt/` 에 `.txt` 가 하나뿐이면 모든 단위가 그것을 사용.
- **장수는 프롬프트에 기재**(예: "3:4 6 장 + 1000×1000 1 장"). 장수 변경은 프롬프트 본문만 수정, 명령은 그대로.
- 결과물은 `output/batch-<타임스탬프>/<이미지 이름>/01.png 02.png …`, 로그는 해당 폴더의 `.logs/`. 스크립트 종료 시 크기 자체 점검 실행.

### 방법 B: codex 에게 맡기기 (자동, 질문 없음)
이미지를 `input/`, 프롬프트를 배치한 뒤 이 폴더에서 codex 를 실행하고 이렇게 말합니다:
> temu-batch-imagegen 스킬로 이미지를 생성해 줘

codex 는 **`input/` 안의 모든 단위**(단일 이미지 + 폴더, 질문 없음)를 처리하고, 같은 규칙으로 `output/batch-<타임스탬프>/<단위 이름>/` 에 생성·저장합니다.

> 상품이 하나여도 동일하게 `bash batch.sh` —— 스크립트가 그 하나만 처리합니다. 별도 명령 불필요.

## 4. 구성

```
temu-batch-imagegen/
├── README.md           # 중국어 간체 (기본)
├── README_KO.md        # 본 파일
├── SKILL.md            # codex 실행 흐름 (codex 가 읽음)
├── 批量出图.command     # 더블클릭 실행기 (macOS, 비개발자용)
├── batch.sh            # 일괄 생성: 이미지마다 독립 세션, 3 병렬 + 직렬 폴백
├── input/              # [단일 이미지] 또는 [이미지 폴더] 배치 (.png/.jpg/.webp, 혼합 가능)
│   ├── dress.png       #   단일 이미지 = 한 상품
│   └── jeans/          #   폴더 = 한 상품의 이미지 세트 (다각도 참조)
├── prompt/             # 창작 프롬프트 (.txt; 이름으로 매칭, 범용은 默认.txt)
│   └── 默认.txt
├── output/             # 결과물: batch-<타임스탬프>/<단위 이름>/01.png 02.png …
└── example/            # 실행 가능한 샘플 (실제 프로젝트와 동일 구성: input + prompt)
    ├── input/product.png
    └── prompt/默认.txt
```

## 5. 프롬프트 작성 팁

- **자연어로** 브리프처럼 "상품 / 스타일 / 요구사항" 을 명확히(`prompt/默认.txt` 가 실제 예시).
- 이미지에 외국어 카피(일본어 셀링 포인트 등)를 넣고 싶으면 프롬프트에 그렇게 쓰면 됩니다. 예: "자연스럽고 고급스러운 일본어 카피".
- **모든 요구사항을 명시**: 크기, 색 재현, 스타일, 얼굴 노출 여부, 배경, 브랜드 로고 처리…… 상품과 플랫폼에 맞게 지정하세요. 도구는 프롬프트대로만 실행하며 **임의의 제약을 가정하지 않습니다**.
- 각 이미지가 **서로 다른** N 장 세트를 원하면 프롬프트에 "각 이미지를 독립 생성하고 포즈/장면 등을 변화" 라고 명시해 "같은 이미지에 글자만 다름" 을 방지.

## 6. 주의

- **이미지 위 외국어에 오타가 날 수 있습니다**: 이미지 모델은 외국어 렌더링에서 가끔 철자를 틀립니다. 각 결과물을 확인하고, 마음에 안 드는 한 장은 codex 에게 **그 한 장만 재생성** 하게 하세요. 100% 정확해야 하는 텍스트는 먼저 깨끗한 이미지를 생성하고 나중에 다른 도구로 카피를 정확히 덧입히는 것이 안전합니다.
- 중간 파일은 codex 기본 디렉터리 `~/.codex/generated_images/` 에 남고, 최종 결과물은 본 프로젝트의 `output/` 에 있습니다.

---

*생성은 codex 내장 `image_gen`(당신의 구독)으로 동작하므로 관리할 API 키가 없습니다. 본 저장소는 macOS 버전입니다.*
