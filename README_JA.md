# TEMU 商品画像 一括生成ツール

`input/` に商品入力をまとめて入れるだけ。各商品は **1 枚の画像** でも、**1 つのフォルダ（同一商品の複数アングル参照画像）** でもかまいません。マッチした創作プロンプトに従い、**codex 内蔵の `image_gen`**（あなたの codex サブスクで動作、**API キー不要**）で、各商品に N 枚の EC メイン画像 / 画像セットを一括生成し、`output/` に保存します。

用途：実写の商品写真をまとめて、指定したスタイルで TEMU / 越境 EC / ネットショップ向けのメイン画像 + 画像セットに展開する。

---

## 1. 準備（初回のみ）

必要なのは **ログイン済みの codex CLI** だけ（codex サブスクがあれば十分）：

```bash
codex --version      # バージョンが表示されれば OK
codex login          # 未ログインなら指示に従ってログイン
```

Python も `OPENAI_API_KEY` も追加インストールも不要です。

---

## 2. codex に組み込む

`temu-batch-imagegen/` フォルダごと codex の skills ディレクトリに入れます：

```bash
cp -R temu-batch-imagegen ~/.codex/skills/
```

> skills ディレクトリに入れず、本フォルダに `cd` して直接使うこともできます（「方法 B」参照）。

---

## 3. 使い方

> 基本ルール：**`input/` 内のすべてのユニット（単一画像 + 画像フォルダ）が処理されます。「どれを / どのプロンプトで / 何枚」とは聞かれません** ——この 3 つは確定済み（全処理・名前でプロンプト照合・枚数はプロンプト本文参照）。

### 最も簡単：`批量出图.command` をダブルクリック（非技術ユーザー向け）
`input/` に画像を入れ、**`批量出图.command` をダブルクリック** するだけ——ターミナルが自動で開き、完了後に `output/` フォルダが表示されます。
- `input/` が空なら案内を表示して input フォルダを開きます。codex 未インストール / 未ログインなら明確にエラーになります。
- 初回ダブルクリックで macOS にブロックされたら、**右クリック → 開く** で一度許可します。
- 前提は同じく **codex のインストール + ログイン**。

### 方法 A：1 コマンドで一括（ターミナルユーザー推奨）
1. **入力**を `input/` に置きます。2 形態を混在可：
   - **単一画像**（`dress.png`）= 1 商品、1 枚の参照画像；
   - **フォルダ**（`jeans/`、同一商品の複数写真）= 1 商品の**画像セット**。中の複数枚が多アングル参照として同一セッションに渡されます。
2. プロンプトを用意（`prompt/*.txt`、要件を自然な文章で記述）：汎用は `默认.txt`。特定商品をカスタムするには、**画像名またはフォルダ名と同名**の `.txt` を作成。
3. 本フォルダ内で実行：

```bash
bash batch.sh            # デフォルトで最大 3 並列
MAX=5 bash batch.sh      # 速くしたいなら並列数を上げる（codex のレート制限に注意）
```

- **1 画像 1 プロセス**：各画像が独立した codex セッションを持ち、互いに干渉しません。デフォルト 3 並列、**失敗は自動で直列リトライにフォールバック**。
- **プロンプト自動照合**：各ユニットは**名前**（画像名 / フォルダ名）で `prompt/<名前>.txt` を探します。同名がなければ `prompt/默认.txt`。`prompt/` に `.txt` が 1 つだけなら全ユニットがそれを使用。
- **枚数はプロンプトに記述**（例「3:4 を 6 枚 + 1000×1000 を 1 枚」）。枚数変更はプロンプト本文だけ編集、コマンドは不変。
- 成果物は `output/batch-<タイムスタンプ>/<画像名>/01.png 02.png …`、ログは同フォルダの `.logs/`。スクリプト終了時にサイズ自己チェックを実行。

### 方法 B：codex に任せる（自動・無質問）
画像を `input/`、プロンプトを配置したら、本フォルダで codex を起動し、こう伝えます：
> temu-batch-imagegen スキルで画像を生成して

codex は **`input/` 内の全ユニット**（単一画像 + フォルダ、質問なし）を処理し、同じルールで `output/batch-<タイムスタンプ>/<ユニット名>/` に生成・保存します。

> 商品が 1 つでも同じく `bash batch.sh` ——スクリプトがその 1 つだけ処理します。専用コマンド不要。

## 4. 構成

```
temu-batch-imagegen/
├── README.md           # 簡体中国語（デフォルト）
├── README_JA.md        # 本ファイル
├── SKILL.md            # codex 実行フロー（codex が読む）
├── 批量出图.command     # ダブルクリック起動（macOS、非技術ユーザー向け）
├── batch.sh            # 一括生成：画像ごとに独立セッション、3 並列 + 直列フォールバック
├── input/              # [単一画像] または [画像フォルダ] を置く（.png/.jpg/.webp、混在可）
│   ├── dress.png       #   単一画像 = 1 商品
│   └── jeans/          #   フォルダ = 1 商品の画像セット（多アングル参照）
├── prompt/             # 創作プロンプト（.txt；名前で照合、汎用は 默认.txt）
│   └── 默认.txt
├── output/             # 成果物：batch-<タイムスタンプ>/<ユニット名>/01.png 02.png …
└── example/            # 実行可能サンプル（実プロジェクトと同じ構成：input + prompt）
    ├── input/product.png
    └── prompt/默认.txt
```

---

## 5. プロンプト作成のコツ

- **自然な文章で**、ブリーフのように「商品 / スタイル / 要件」を明記（`prompt/默认.txt` が実例）。
- 画像に外国語コピー（日本語の訴求点など）を載せたいなら、プロンプトでそう書くだけ。例「自然で上質な日本語コピー」。
- **すべての要件を明示**：サイズ、色再現、スタイル、顔出しの有無、背景、ブランドロゴの扱い……商品とプラットフォームに合わせて指定を。ツールはプロンプト通りに実行し、**勝手な前提を置きません**。
- 各画像が**それぞれ異なる** N 枚セットが欲しいなら、プロンプトに「各画像を独立生成し、ポーズ/シーン等を変える」と明記し、「同じ画像で文字違い」を防止。

## 6. 注意

- **画像上の外国語に誤字が出ることがあります**：画像モデルは外国語の描画で時々スペルを誤ります。各成果物を確認し、気に入らない 1 枚は codex に**その 1 枚だけ再生成**させてください。100% 正確が必須の文字は、まずクリーンな画像を生成し、後から別ツールでコピーを正確に重ねるのが安全です。
- 中間ファイルは codex のデフォルトディレクトリ `~/.codex/generated_images/` に残り、最終成果物は本プロジェクトの `output/` にあります。

*生成は codex 内蔵の `image_gen`（あなたのサブスク）で動作し、API キー管理は不要です。本リポジトリは macOS 版です。*
